в начало

Немного про git

Список команд для работы с гитом:

Переход в нужную папку репозиторий

cd/...

Установка конфигураций (--local -локально, --global -глобально)

git config user.name "Stas" user email ..@gmail.com

смотрим статус / файлы проекта

git status

Работа с индексом

Добавляем файлы в индекс

git add

Добавляем все файлы в индекс

git add -A

Добавляем в индекс все файлы в текущей папке

git add.

добавляем конкретный файл в индекс

git add style.css

Работа с Коммитами

Создаем коммит с описанием со всеми файлами

git commit -a -m"first commit"

смотрим логи коммитов

git log

Работа с ветками

создаем новую ветку

git branch new-branch

Переключаемся на новую ветку

git checkout new-branch

Содание и переключение на новую ветку одной командой

git checkout -b new-branch

Производит слияние 2-х веток. 

git checkout master
git merge new-branch

Работа с Удаленными репозиторийми

отправляем файлы в определенную ветку репозитория

git push -u -origine master

связывается с удалённым репозиторием и забирает из него все изменения, которых у вас пока нет и сохраняет их локально

git fetch

работает как комбинация команд git fetch и git merge, т. е. Git вначале забирает изменения из указанного удалённого репозитория, а затем пытается слить их с текущей веткой

git pull

клонируем удаленный репозиторий себе на компьютер в конкретную папку

git clone https://... it projects

Отмена действий

git restore
