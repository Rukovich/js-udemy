Async, defer, Динамические скрипты

defer

в конструкции ниже:

|script> defer src="js/script.js">

Можно поместить в тег head, defer позволяет скрипту запускаться в фоновом режиме и вызывать его тогда, когда скрипт понадобится

Особенности:

  • Они никогда не блокируют страницу
  • Всегда дожидаются загрузки верстки

async

дан в конструкции ниже:

script> async src="js/script.js">

Особенности:

  • Страница не ждет асинхронных скриптов, страница просто обрабатывается и отображается
  • события DOM и асинхронные скрипты не ждут друг друга

Простыми словами скрипт запускается в фоновом режиме и запускается тогда когда был загружен, не дожидаясь остальных скриптов

Вывод: Когда используем скрипты с атрибутом async мы должны быть точно уверены, что этот скрипт не зависит от DOM структуры и других скриптов

Добавляем скрипт на страницу напряму через script.js:

Чтобы добавить скрипт на страницу напрямую можно воспользоваться следующей конструкцией:

function loadScript(src) {
  const script = document.createElement('script');
  script.src = src;
  script.async = false;
  document.body.append(script);
}

loadScript('js/test.js');
loadScript('js/some.js');
